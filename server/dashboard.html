Overview

LocalGo is a private, local URL shortener that runs entirely on your machine. Map short keywords to full URLs — type go/docs in your browser and navigate there instantly.

Unlike cloud-based link shorteners, LocalGo has no accounts, no server costs, and no data leaving your machine. It's a Node.js server you start locally and a Chrome extension that intercepts go/* navigation.

How it works: Chrome intercepts requests to http://go/*, looks up the keyword in your local link store, and issues a 302 redirect to the full destination. Everything happens in milliseconds, entirely on your device.

Creating Go Links

Use the form in the header bar to create a new link. Press Enter from any field, or click Create.

go/keyword

The short name you'll type in the address bar. Lowercase letters, numbers, and hyphens. Supports {var} placeholders for template links.

Destination URL

The full URL to redirect to, starting with https://.

Tags

Comma-separated labels for organizing links. Example: work, docs, jira. Tags appear in the sidebar for filtering.

Aliases (second row)

Additional keywords that resolve to the same link, comma-separated. Example: keyword gmail with aliases mail, inbox.

Expires (second row)

Optional date after which the link stops working. Leave blank for a permanent link.

Hover any link row to reveal copy, edit, and delete actions on the right.

Template Links

A single go link can handle many URLs by accepting path variables. Use {varname} placeholders in both the keyword and destination URL.

Keywordjira/{id}
URLhttps://jira.company.com/browse/{id}
UsageType go/jira/PROJ-123 → opens that Jira ticket

Multiple variables work too:

Keywordgh/{user}/{repo}
URLhttps://github.com/{user}/{repo}

Variable values are URL-encoded automatically. Each placeholder matches one path segment (no slashes). Variables appear highlighted in amber in the link list.

Aliases

Aliases give a single link multiple names without duplicating the entry. They appear as italic =alias chips on the link row.

Keywordgmail
Aliases=mail, =inbox, =email
go/gmail, go/mail, go/inbox, go/email — all work.

Type aliases comma-separated in the Aliases field. The go/ prefix is stripped automatically if included. Aliases are searchable in the dashboard.

Expiring Links

Set an expiry date to automatically deactivate a link after a deadline — useful for event pages, promotional URLs, or temporary access.

  • Links expire at 23:59 on the selected date (local time).
  • An expires soon badge appears when expiry is within 7 days.
  • Visiting an expired link shows a banner and opens this dashboard to edit it.
  • An expired badge marks the row.
  • The Expired filter in the sidebar shows all expired links at once.
  • Edit the link and click clear next to the date to reactivate it.

Tags & Filtering

Tags are free-form labels for organizing and filtering your links. They work alongside the search bar and sidebar filters.

Adding tags: Enter comma-separated tags in the Tags field when creating or editing. Tags are stored lowercase.

Tag sidebar: Every unique tag appears in the left panel. Click a tag to filter the list to matching links; click All links to clear.

Search: The search bar matches across keyword, destination URL, tags, and aliases simultaneously.

Bulk tagging: Tick checkboxes on multiple rows, then click + Add tag in the bulk action bar to tag them all at once.

Link Health

LocalGo tracks how fresh and alive your links are. Status badges appear on each row automatically.

dead

The destination returned an error (4xx / 5xx) or timed out during a health check. Click ⬡ Check all links in the sidebar to run checks across all your links at once.

expires soon

The link has an expiry date within the next 7 days.

expired

The expiry date has passed. The link no longer redirects.

stale

Not used in 90 days (or not used since creation over 90 days ago). Not broken — just a nudge to review old links.

Template links (containing {var}) are skipped during health checks since their URL varies per request.

Import & Export

Back up your links or migrate from another tool using the JSON import/export buttons in the sidebar.

↓ Export JSON

Downloads all your links as golinks.json. Use for backups or copying links to another machine.

↑ Import JSON

Select a JSON file to merge into your link store. If a keyword already exists, the imported entry overwrites it. Visit counts and creation dates in the file are preserved.

Minimum required format:

[{
  "short_path": "docs",
  "long_url": "https://docs.example.com",
  "tags": ["work"],
  "hits": 42
}]

Browser Setup

Hosts file (recommended — enables go/keyword directly in address bar)

  1. Open Notepad as Administrator.
  2. Open C:\Windows\System32\drivers\etc\hosts
  3. Add at the bottom: 127.0.0.1 go
  4. Save, then reload the LocalGo extension at chrome://extensions.

Omnibox fallback (no hosts file needed)

Type go in Chrome's address bar and press Tab. The bar switches to LocalGo mode — type your keyword and press Enter to navigate.

Extension popup

Click the LocalGo icon in the Chrome toolbar. The popup pre-fills the destination URL from the current page. Enter a keyword and click Create to save instantly.

Starting the server

The local server must be running for go links to resolve. Start it from the project folder:

node server/server.js

The server runs on port 2999, accessible only from your own machine (127.0.0.1). To start automatically on boot, add it to Windows Task Scheduler or create a startup shortcut.

AI Features

LocalGo uses Chrome's built-in Gemini Nano for optional AI features. The model runs entirely on your device — no prompts or page content are sent to Google's servers.

Requirements: Chrome 138 or later. Chrome automatically downloads the Nano model (~4 GB) in the background on first use. Check status at chrome://on-device-internals. If Nano isn't available, all AI features are silently hidden with no errors.

Auto-tagging & keyword suggestion (extension popup)

When you open the popup on a webpage, Gemini Nano analyses the page title, URL, meta description, and first heading to suggest a go-link keyword and up to 3 relevant tags.

  • An "AI suggests" strip appears once the model responds (~1–2 seconds).
  • Click the keyword button to apply it to the keyword field.
  • Click any tag chip to add it — already-added chips dim and disable.
  • All suggestions are optional — type your own and ignore them.
  • Automatically hidden on chrome:// pages.

Summary peek (dashboard)

In the link list, hover any destination URL for 600 ms. LocalGo fetches a text snapshot of the target page (via the local server, to avoid browser CORS restrictions) and summarises it using Nano's Summarizer API.

  • Summaries are cached for the session — hovering the same link again is instant.
  • Move the mouse away to dismiss the tooltip.
  • Pages that error, time out, or return no text silently skip the tooltip.
